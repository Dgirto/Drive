¿Qué es y para qué sirve?

El conector Drive permite que el agente de Ruvic liste, lea, descargue y suba archivos en el Google Drive de uno o varios usuarios de tu dominio Google Workspace, usando una sola credencial administrada centralizadamente por tu equipo de TI — sin que cada usuario tenga que autorizar su propio Drive por separado. Una vez configurado, puedes pedirle al agente cosas como:

  • "Busca en el Drive de facturación los archivos que contengan 'factura' en el nombre."
  • "Descarga el último reporte mensual que subió soporte@empresa.com."
  • "Sube este archivo a la carpeta de reportes de cobranzas@empresa.com."

Operaciones soportadas:

Operación Qué hace
file.list Busca/lista archivos usando filtros (sintaxis de búsqueda de Drive)
file.read Lee los metadatos de un archivo (nombre, tipo, dueño, tamaño, etc.)
file.download Descarga el contenido de un archivo a disco local
file.upload Sube un archivo local al Drive del usuario (máx. 20 MB)
Solo para Google Workspace Este conector usa delegación de dominio, una función exclusiva de Google Workspace. No funciona con cuentas de Google personales (@gmail.com).

Prerrequisitos

  • Una organización con Google Workspace (no funciona con cuentas personales @gmail.com).
  • Acceso de superadministrador de Google Workspace — es la única persona que puede autorizar la delegación de dominio (paso único, sección 3).
  • Acceso al Google Cloud Console del mismo dominio, con permisos para crear un proyecto y una cuenta de servicio.
  • Los correos de los usuarios cuyo Drive va a usar el agente (ej. facturacion@tuempresa.com).
¿Ya configuraste Gmail Workspace? Puedes reutilizar la misma cuenta de servicio del conector Gmail (Google Workspace) en lugar de crear una nueva — solo hay que agregar el scope de Drive a la misma entrada de delegación en el Admin Console.

Configuración en el fabricante

Vas a crear una cuenta de servicio en Google Cloud y autorizarla mediante delegación de dominio para que pueda actuar en nombre de cualquier usuario autorizado. Esto se hace una sola vez por dominio — después, agregar un usuario nuevo no requiere repetir estos pasos, solo compartírselo al agente por su correo.

  1. Crea (o selecciona) un proyecto en Google Cloud Console.

    Ve a console.cloud.google.com/projectcreate y crea un proyecto (o reutiliza uno existente del mismo dominio).

  2. Habilita la Google Drive API.

    En el menú APIs y servicios → Biblioteca, busca "Google Drive API" y pulsa Habilitar.

  3. Crea la cuenta de servicio (o reutiliza la de Gmail Workspace).

    En IAM y administración → Cuentas de servicio → Crear cuenta de servicio. Dale un nombre (ej. ruvic-drive) y termina el asistente sin asignar roles adicionales de proyecto.

  4. Genera y descarga la clave JSON.

    Abre la cuenta de servicio → pestaña Claves → Agregar clave → Crear clave nueva → JSON. Se descargará un archivo .json: guárdalo, lo usarás en el formulario de Ruvic.

    [ Captura: pestaña Claves de la cuenta de servicio con el botón "Agregar clave" — reemplazar por assets/img/drive-sa-key.png ]
  5. Anota el ID de cliente único de la cuenta de servicio.

    En la pestaña Detalles de la cuenta de servicio, copia el ID de cliente único (una cadena numérica larga). Lo necesitarás en el siguiente paso.

  6. El superadministrador autoriza la delegación de dominio (paso crítico).

    Este paso solo lo puede hacer el superadministrador de tu organización de Google Workspace. Ve a admin.google.com → Seguridad → Control de acceso a las API → Delegación en todo el dominio. Si el Client ID ya está agregado (ej. por Gmail Workspace), edítalo y agrega el scope de Drive; si no, pulsa Agregar nuevo e ingresa:

    • ID de cliente: el ID de cliente único copiado en el paso anterior.
    • Alcances OAuth: 
      https://www.googleapis.com/auth/drive

    Guarda. Sin este paso, cualquier intento de conexión falla con el error unauthorized_client.

    [ Captura: pantalla de delegación en todo el dominio con el ID de cliente y los scopes — reemplazar por assets/img/drive-domain-delegation.png ]
  7. Define qué usuario usarás para la prueba inicial.

    Anota el correo de un usuario del dominio (ej. facturacion@tuempresa.com) para el formulario de Ruvic. Después de configurado, el agente podrá usar cualquier otro usuario autorizado del mismo dominio, no solo este.

Permisos requeridos

La cuenta de servicio necesita el scope completo de Drive para poder listar, leer, descargar y subir archivos.

Scope Para qué lo usa el conector
drive Buscar y listar archivos (file.list), leer metadatos (file.read), descargar (file.download) y subir archivos (file.upload)
Este scope da acceso completo de lectura y escritura A diferencia de scopes más restringidos como drive.file (solo archivos creados por la app) o drive.readonly (solo lectura), drive permite ver, descargar, modificar y subir cualquier archivo al que el usuario impersonado tenga acceso, incluidas unidades compartidas. La delegación aplica a todo el dominio: la cuenta de servicio puede impersonar a cualquier usuario autorizado, incluidos los que se creen en el futuro.

Configuración en Ruvic

Entra a Settings → Conectores, busca la tarjeta Drive (Google Workspace — cuenta de servicio) y pulsa Configurar. Diligencia el formulario con los valores que obtuviste en la sección anterior:

Campo del formulario Valor
JSON de la cuenta de servicio Abre el archivo .json descargado en el paso 4 y pega todo su contenido
Usuario de prueba El correo definido en el paso 7 (ej. facturacion@tuempresa.com)
Timeout de solicitud Opcional; 30 segundos por defecto

Guarda y pulsa Probar conexión — la plataforma se conectará a Drive API impersonando al usuario de prueba y confirmará que la delegación de dominio funciona.

[ Captura: formulario del conector Drive en Settings → Conectores — reemplazar por assets/img/drive-form.png ]

Verificación

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: "Conexión exitosa al Drive de facturacion@tuempresa.com (delegación de dominio)". La tarjeta del conector cambia a estado Configurado.

  2. Automatización mínima de prueba, incluido un segundo usuario.

    Abre una conversación nueva con el agente y pídele:

    "Lista los últimos 5 archivos del Drive de facturacion@tuempresa.com."

    Luego prueba con otro usuario del dominio distinto al de la prueba inicial (ej. soporte@tuempresa.com) — si el agente responde con datos reales de ambos sin pedir nueva configuración, la delegación de dominio quedó operativa de punta a punta.

Solución de problemas

Mensaje / síntoma Causa probable y solución
"unauthorized_client" al probar la conexión El error más común. El conector te dirá exactamente qué revisar: 1) que el Client ID esté agregado en el Admin Console (paso 6 de la sección anterior), 2) que el scope incluya exactamente https://www.googleapis.com/auth/drive, 3) que el usuario de prueba pertenezca al mismo dominio Workspace. Solo el superadministrador puede corregir el punto 1.
"La Google Drive API no está habilitada en el proyecto de Google Cloud…" Falta habilitar la Google Drive API en el proyecto (paso 2 de la sección anterior). El mensaje del conector incluye el enlace directo de activación; habilítala y espera 1-2 minutos.
"…no contiene un JSON válido" Se pegó un fragmento del archivo o se modificó el contenido. Vuelve a abrir el .json descargado y pega todo su contenido, sin editarlo.
Funciona con el usuario de prueba pero falla con otro usuario del dominio Verifica que ese otro usuario exista y esté activo en el mismo dominio Workspace — la delegación cubre todo el dominio automáticamente, pero un correo mal escrito o de otro dominio siempre fallará.
"Drive API no respondió correctamente (HTTP 429)" Límite de cuota temporal de la API. Es transitorio; el agente puede reintentar en unos segundos.
El conector aparece "Sin configurar" tras guardar Algún campo obligatorio quedó vacío. Ábrelo de nuevo y completa el JSON de la cuenta de servicio y el usuario de prueba.

Preguntas frecuentes y notas de seguridad

¿Qué tan sensible es el JSON de la cuenta de servicio?

Es la credencial de mayor cuidado de este conector. El scope drive da acceso completo de lectura y escritura a cualquier archivo al que el usuario impersonado tenga acceso — no solo a los archivos creados por el conector. Trátala con el mismo cuidado que la contraseña de un administrador — nunca la compartas por chat, correo o capturas de pantalla.

¿Puedo usar la misma cuenta de servicio que Gmail Workspace?

Sí. Una cuenta de servicio puede tener varios scopes autorizados a la vez en la misma entrada de delegación del Admin Console. Solo agrega https://www.googleapis.com/auth/drive a la lista de scopes existente, separado por coma, sin espacios extra.

¿El agente puede eliminar o sobrescribir archivos?

El conector solo expone las operaciones file.list, file.read, file.download y file.upload — no expone eliminación ni sobrescritura. El scope subyacente sí lo permitiría técnicamente, pero el código generado no incluye esas operaciones a menos que se pidan explícitamente y se extienda el conector.

Rotación de credenciales

Recomendado cada 90 días o según tu política: genera una nueva clave JSON en Google Cloud Console (la cuenta de servicio puede tener varias claves activas a la vez), actualízala en el formulario del conector, prueba la conexión, y luego elimina la clave anterior. La delegación de dominio no cambia — sigue asociada al mismo Client ID.

¿Qué hago si el JSON se compromete?

  1. El superadministrador elimina o edita la entrada de delegación en el Admin Console (Seguridad → Delegación en todo el dominio) — esto revoca el acceso de inmediato.
  2. Elimina la clave comprometida en Google Cloud Console (IAM y administración → Cuentas de servicio → Claves).
  3. Genera una nueva clave, vuelve a autorizar la delegación con el mismo Client ID, y actualiza el formulario del conector.
Buenas prácticas Usa una cuenta de servicio dedicada (o compartida solo con Gmail Workspace); nunca compartas el JSON por canales no seguros; y revisa periódicamente en el Admin Console qué Client IDs tienen delegación de dominio activa en tu organización y con qué scopes.
